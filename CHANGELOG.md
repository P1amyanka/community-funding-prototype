# Changelog

## Prototype v0.3 / allocation-v0.2 — 2026-07-22

### Інтерфейс
- Головну сторінку перейменовано на «Створіть ініціативу».
- Оновлено пояснення процесу для менеджера та учасників.
- Поля «Опис» і «Бюджет» отримали нові приклади.
- Після створення ініціативи одразу відкривається кабінет менеджера.
- Прибрано проміжний екран «Ініціативу створено».
- Сторінку керування перейменовано на «Кабінет менеджера».
- Технічні статуси замінено зрозумілими користувацькими формулюваннями.
- Прибрано пояснення внутрішньої роботи алгоритму й текст про автоматичне закриття.
- Дедлайн не показується, якщо його не встановлено.
- Для учасника залишено одне поле: «Максимальна сума внеску» без додаткової підказки.
- Поточний візуальний стиль збережено.

### Алгоритм allocation-v0.2
- Прибрано мінімальну суму, яку раніше вводив учасник.
- Кожен учасник задає лише максимальну суму внеску.
- Система розраховує найменший необхідний внесок для кожного за принципом рівномірного розподілу з індивідуальними верхніми межами.
- Якщо рівна частка перевищує максимум учасника, його внесок фіксується на максимумі, а залишок порівну перерозподіляється між іншими.
- Розрахунок виконується в копійках, а залишок від округлення розподіляється по одній копійці.
- Якщо сума максимумів менша за ціль, система показує дефіцит.
- CSV оновлено: максимум, розрахований внесок, версія алгоритму.

### Сумісність
- Для сумісності з чинним Supabase RPC у `p_min_amount` передається `0`; нове поле мінімуму в інтерфейсі відсутнє.
